# MEDIA-708: Orval Client Generation Pipeline

## Story

**Epic:** MEDIA-FE-ADMIN — Admin Frontend  
**Priority:** High (blocks all API calls from frontend)  
**Effort:** 2 points  
**Status:** ⏳ Planned  
**Depends on:** MEDIA-700 (project setup), MEDIA-712 (OpenAPI spec)

---

## Summary

Set up Orval to auto-generate a fully typed TypeScript API client with TanStack Vue Query hooks from the OpenAPI spec. This replaces all hand-written `fetch()` calls with generated, type-safe composables like `useGetQueue()`, `useAddToQueue()`, etc.

---

## Why Orval?

| Without Orval | With Orval |
|--------------|-----------|
| Hand-written types for every DTO | Types auto-generated from spec |
| Manual `fetch()` with error handling | Generated TanStack Query hooks |
| Types drift from API silently | Build fails if spec changes |
| ~500 lines of API client code | ~20 lines of config |

## Orval Configuration

```typescript
// orval.config.ts (frontend root)
import { defineConfig } from 'orval'

export default defineConfig({
  mediaPlatform: {
    input: {
      target: '../../openapi.json', // Generated by MEDIA-712 (relative to frontend/)
    },
    output: {
      target: './src/generated/api',
      schemas: './src/generated/models',
      client: 'vue-query',
      mode: 'tags-split',          // Split by endpoint tags (queue, player, admin, etc.)
      override: {
        mutator: {
          path: './src/lib/api-client.ts',
          name: 'customFetch',
        },
        query: {
          useQuery: true,
          useMutation: true,
          signal: true,             // AbortController support
        },
      },
    },
  },
})
```

## Custom Fetch Mutator

Orval's generated hooks use a custom fetch wrapper that handles auth tokens and base URL:

```typescript
// src/lib/api-client.ts
import { useAuthStore } from '@/stores/useAuthStore'

const API_BASE = import.meta.env.VITE_API_BASE_URL || '/api'

export const customFetch = async <T>(config: {
  url: string
  method: string
  data?: unknown
  params?: Record<string, string>
  headers?: Record<string, string>
  signal?: AbortSignal
}): Promise<T> => {
  const authStore = useAuthStore()
  const headers: Record<string, string> = {
    'Content-Type': 'application/json',
    ...config.headers,
  }

  // Inject Bearer token if authenticated
  const token = authStore.token
  if (token) {
    headers['Authorization'] = `Bearer ${token}`
  }

  const url = new URL(`${API_BASE}${config.url}`)
  if (config.params) {
    Object.entries(config.params).forEach(([key, value]) =>
      url.searchParams.append(key, value)
    )
  }

  const response = await fetch(url.toString(), {
    method: config.method,
    headers,
    body: config.data ? JSON.stringify(config.data) : undefined,
    signal: config.signal,
  })

  if (!response.ok) {
    const error = await response.json().catch(() => ({ detail: response.statusText }))
    throw new ApiError(response.status, error)
  }

  // Handle 204 No Content
  if (response.status === 204) return undefined as T
  return response.json()
}
```

## Package.json Scripts

```json
{
  "scripts": {
    "generate:api": "orval",
    "generate:api:watch": "orval --watch",
    "prebuild": "pnpm run generate:api",
    "dev": "pnpm run generate:api && vite"
  }
}
```

- `generate:api` runs before `build` and `dev`
- `generate:api:watch` watches `openapi.json` for changes during development
- Generated files are **not** committed to git (in `.gitignore`)

## Usage in Components

```vue
<script setup lang="ts">
import { useGetQueue, useAddToQueue } from '@/generated/api/queue/queue'
import type { AddToQueueRequest } from '@/generated/models'

// GET /queue — auto-refreshes, cached by TanStack Query
const { data: queue, isLoading, error } = useGetQueue()

// POST /queue/add — mutation with optimistic update
const { mutate: addToQueue, isPending } = useAddToQueue({
  mutation: {
    onSuccess: () => {
      // TanStack Query auto-invalidates related queries
    },
  },
})

function handleAdd(request: AddToQueueRequest) {
  addToQueue({ data: request })
}
</script>
```

## MSW Integration (Testing)

Orval generates Mock Service Worker handlers for every endpoint:

```typescript
// In vitest setup
import { setupServer } from 'msw/node'
import { getQueueMockHandler } from '@/generated/api/queue/queue.msw'

const server = setupServer(...getQueueMockHandler())

beforeAll(() => server.listen())
afterEach(() => server.resetHandlers())
afterAll(() => server.close())
```

## Notes

- The `openapi.json` file is the single source of truth — change the API, regenerate, done
- Generated files should never be manually edited
- If a new endpoint is added, just re-run `pnpm run generate:api`
- The `vue-query` client generates composables, not raw fetch calls
